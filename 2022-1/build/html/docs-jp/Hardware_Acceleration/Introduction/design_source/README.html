UG1352: Alveo を使用する開発¶

このリポジトリには、『Alveo を使用する開発』 (UG1352) の演習および Vitis 入門チュートリアルの「ランタイムおよびシステム最適化」のソース ファイルが含まれます。

Git サブモジュールを使用して、Vitis ビジョン ハードウェア アクセラレーション ライブラリなどのその他のリポジトリを含めます。このリポジトリを正しくクローンするには、--recurse-submodules コマンドライン オプションを含める必要があります。

git clone --recurse-submodules https://github.com/Xilinx/Vitis-In-Depth-Tutorial.git

このリポジトリには、ハードウェアおよびソフトウェアの両方のソースが含まれています。

このリポジトリは、Vitis 統合開発環境の 2020.1 リリースを使用して、次のプラットフォームに対してハードウェア検証されています。

Alveo カード プラットフォーム XRT
U200 xilinx_u200_xdma_201830_2 2.6.655
U250 xilinx_u250_xdma_201830_2 2.6.655
U50 xilinx_u50_gen3x16_xdma_201920_3 2.6.655

ハードウェア デザインを構築する¶

このデザインのすべてのハードウェア ソースは hw_src ディレクトリの下にあり、 make を実行すると構築できます。その前に、config.mk を編集するか、特定のシステムを指定するようにコマンド ラインでこれらの値を上書きします。

# Options for TARGET: sw_emu, hw_emu and hw
TARGET ?= hw
# Options for DEVICE: u200, u250. Default platform is XDMA, defined by PLATFORM
DEVICE ?= u250
# If other some specific platforms needs to be used, provide platform path directly
PLATFORM ?= xilinx_$(DEVICE)_xdma_201830_2
# If your platform is not in the standard install area edit this line
PLATFORM_REPO_PATH ?= /opt/xilinx/platforms/

この後、プラットフォームをビルドします。

cd hw_src make

または、コマンド ラインで特定のプラットフォームを指定する場合は、次のコマンドを実行します。

cd hw_src PLATFORM=xilinx_u50_gen3x16_xdma_201920_3 make

プラットフォーム名は必要に応じて変更します。

ソフトウェア デザインの構築¶

これらのリファレンス用のソフトウェアはすべて CMake を使用して構築されています。一部の例では、外部ライブラリ、特に OpenCV と OpenMP に依存しています。これらのライブラリがシステムに存在しない場合、依存関係のある例は構築されません。

このアーカイブのルート ディレクトリからソフトウェアを構築します。

mkdir build cd build cmake .. make -j

注記: これは、.xclbinファイルが存在する、ハードウェア構築後に実行する必要があります。

結果の例を実行する場合は、『Alveo を使用する開発』 (UG1352) またはそれに付随するチュートリアルに示すように直接実行してください。一部の例 (7 と 8) では、追加のコマンド ライン引数が必要です。コマンド ライン引数を指定せずにこれらの例を実行すると、詳細な指示を含むヘルプ メッセージが表示されます。

./00_load_kernels
